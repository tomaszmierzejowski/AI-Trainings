# Module 3 Exercise — Answer Key (TRAINER ONLY — do not distribute)

## The three planted errors

**Error 1 — Wrong number (findable by arithmetic).**
"falling 15% by June" — actual decline March→June is (1,305 − 1,150) / 1,305 = **11.9%, i.e. ~12%**, not 15%.
*Design:* the classic checkable-but-unchecked figure. Most pairs find this one first; it rewards the "numbers first" spot-check habit.

**Error 2 — Fabricated citation (findable by looking for the source).**
"the 2026 Nordic Service Desk Benchmark (see attached benchmark report)" — **no such document exists in the source pack.** The pack contains exactly two sources: the ops table and the memo.
*Design:* tests the "no citation, no claim — and click the citation" habit. Some pairs mark it suspicious but hesitate to call it fabricated; push in debrief: absence of the source IS the finding.

**Error 3 — Inverted conclusion (the survivor).**
"the data indicates that the resolution process is healthier than at the start of the year" and "the staffing gap… appears to have been absorbed without material operational impact."
Both claims invert the data: **average resolution time rose every single month, 8.2h → 11.2h (+37%)** — the clearest trend in the table — and the memo explicitly provides the staffing cause. The supporting fact used ("escalations down 14% from March peak") is *true*, which is what makes the paragraph persuasive: a real number is deployed to decorate a false conclusion.
*Design:* this is the error that survives longest in nearly every group because the paragraph is fluent, hedged ("indicates," "appears"), and anchored by a genuine statistic. It is the exercise's teaching payload.

## True statements (deliberately included — do not let pairs over-flag)
- March peak at 1,305: true.
- Escalations down 14% from March peak: true (121 → 104 = 14.0%).
- CSAT decline 4.1 → 3.7 flagged as a watch item: true and fairly stated.
- The memo's content is quoted accurately.

## Debrief script (5 min)
1. Poll: which error did each pair find first / last / not at all? (Typical pattern: 1 → 2 → 3, with several pairs missing 3 entirely.)
2. Land the insight: **fluency masks falsehood** — the best-written paragraph carried the worst error, and it recruited a true number as camouflage.
3. Connect to the matrix: this report is high-stakes + easy-to-verify (the source table is right there). The failure mode wasn't ability — it was that the prose *didn't feel like it needed checking*.
4. Close: "The 60-second spot-check order — names, numbers, dates, then conclusions-vs-data — exists precisely because of error 3."

## Facilitation notes
- If a pair finishes early, ask them to also verify the "seasonal pattern" claim — it's unverifiable from the pack even ignoring the fake citation (only 6 months of data; no prior year). Bonus finding.
- If someone argues error 3 is "interpretation, not error": point at the resolution-time column and ask them to defend "healthier" out loud. It collapses quickly, and the moment is memorable.
- Rebuild variant for repeat clients: keep the error *types* (wrong figure / fabricated source / fluent inversion decorated with a true stat) and swap the domain.
