Technical Detail: Confidence Scoring (ECI) & Evidence Discipline

Evidence Coverage Index (ECI):

ECI = 0.50 × Verification Rate + 0.35 × Critical Verification Rate + 0.15 × Completeness

  • Verification Rate = (Verified + Partial) / total findings — measures evidence breadth
  • Critical Verification Rate = Verified among Critical+High / total Critical+High — ensures high-severity findings are proven
  • Completeness = 1 − (Cannot Determine / total) — measures knowledge gaps

Evidence Status Labels: Verified (confirmed by code with file:line citation), Partial (some evidence), Assumed (inferred from patterns), Cannot Determine (unresolvable from codebase).

Confidence Tiers: Very High (90–100%), High (70–89%), Medium (50–69%), Low (30–49%), Very Low (0–29%).

Innovation Paralysis

Talent & Cost Pressure

Available talent pool for EJB, Struts, and WebLogic continues to shrink. Acute key-person risk on critical flows. Onboarding new team members is slow due to undocumented complexity. Maintaining the current platform becomes progressively more expensive.

6

Vendor Lock-in

Platform deeply tied to specific application server and deployment model. Cannot containerize without significant rework. Cannot take advantage of cost-competitive cloud options or adopt Kubernetes without modernization.

Architecture Patterns — Three Coexisting Generations

Legacy EJB Stack

Three-layer stack: *App (GUI + EJB), *WS (SOAP facade), *WSClient (generated JAR from WSDL). App and WS share the same EJB module and DB schema.

Spring/zKoss Apps

Application and REST service in a single deployment. No separate WS/WSClient split. Modern-er pattern but still tightly coupled to shared database schemas.

Central Hub Structure

Single EAR with three co-deployed modules: web UI (WAR), core business logic (EJB for pricing/calculations), and catalogmodule data (EJB). All outbound calls originate from this single deployment.

URL Routing (Browser Integration)

Beyond backend calls, the hub integrates with applications via browser redirects. Context passed in URL parameters. After task completion, target app redirects back. 7+ apps use this pattern.

Architecture Summary & Risk Assessment

Finding: 18 services communicate via point-to-point integrations across 7+ protocols with no API gateway, service mesh, or message broker. The central hub orchestrates calls to nearly all other services and 12 external systems.

  • Hub-and-spoke fragility — Central hub failure cascades to nearly all platform functions. No circuit breakers, retries, or fallback mechanisms exist anywhere in the platform.
  • Legacy technology stack — EJB 1.1/2.0, Struts 1.1 (EOL 2013), Spring 4.3 (EOL 2020), Java 8, Oracle WebLogic. Frameworks are no longer maintained with known security vulnerabilities.
  • Mixed architecture patterns — Three distinct generations coexist: legacy EJB with BMP persistence, Spring/zKoss web apps, and monolithic Struts frontend. No consistent service contract definitions.
  • External system dependencies — 12 external systems (manufacturer APIs, CRM provider, identity provider, finance platform, etc.) with no standardized integration pattern.
  • Shared services as hidden coupling — Document generation and OrderMgmt are called by multiple domains, creating cross-domain dependencies invisible without full integration tracing.
